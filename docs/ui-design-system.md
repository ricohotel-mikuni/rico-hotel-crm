# 大栄商事株式会社 UIデザインシステム v1.0

**ステータス**: 承認済み(2026-07-10)
**根拠**: [大栄商事ERP開発憲章 Ver.1.3](./development-constitution.md) 第十一章
**適用範囲**: 大栄商事ERP全画面(ブランド切替を除きすべての画面で共通)

このファイルは、拠点ダッシュボード(リコホテル三国 NEO TODAY画面)で確立されたUIパターンを、大栄商事ERP全体の標準デザインシステムとして明文化したものである。今後新しい画面を作る際は、個別にデザインを考えるのではなく、本書のトークン・パターンをそのまま使う。ブランド(ロゴ・名称)だけが画面ごとに切り替わり、それ以外(配色・余白・カード・アイコン・フォント・NEO・アニメーション)は全画面で完全に統一する。

## 1. 配色トークン

拠点ダッシュボード画面(`src/modules/hotels/PropertyHub.jsx`)で確立した値。今後、他画面へ展開する際はこの値を `src/lib/constants.js` の `C` 定数へ正式に統合し(ERP開発憲章第11条)、画面ごとのローカル定義を廃止する。

| 用途 | 値 | 備考 |
|---|---|---|
| 背景(ページ) | `#071C3A` | 最も濃いネイビー |
| カード背景 | `#0F2A4D` | ページ背景よりやや明るいネイビー |
| ボーダー | `#1A2A4A` | カードの枠線・区切り線 |
| アクセント(ゴールド) | `#D4AF37` | 見出し・現在地・強調数値・NEOの発光色 |
| 文字(メイン) | `#FFFFFF` | 見出し・主要な数値 |
| 文字(サブ) | `#C7D0E0` | 本文・リスト項目 |
| 文字(補足) | `#8A96AC` | ラベル・キャプション・「準備中」等 |
| 成功・チェックイン系 | `#4CD964` | グリーン |
| 警戒・チェックアウト系 | `#B366FF` | パープル |
| 注意・清掃/夕食系 | `#F59E0B` | オレンジ |
| 情報・駐車場系 | `#3A6DFF` | ブルー |
| 緊急 | `#ff8a7a` | 優先度バッジ「緊急」のみ |

角丸: カード `18px`、カード内の小要素(KPIセル・吹き出し等) `14px`、バッジ・ボタンは `pill`(999px)。
余白: 24px を基準単位とする(カード間・カード内側パディング)。KPIセル等の小要素は 12〜14px。
シャドウ: 暗い背景同士のコントラストが低いため、強い影は使わない。カードは背景色とボーダーの差だけで層を表現する。

## 2. 主要コンポーネントパターン

### 2.1「○○ TODAY」ヒーローカード

各モジュールのホーム画面(拠点ダッシュボードの「NEO TODAY」が最初の実装例)の最上部に置く、そのモジュールのAIサマリーカード。

- 左カラム: キャラクター(NEO、または将来モジュール専用キャラクターがあればそれ)+「◯◯ TODAY」ゴールド見出し + AI総合評価(★+一言、コンパクトに)。
- 右カラム: 補足情報(拠点ダッシュボードでは天気)。**カードの中にさらに別カードを作らない** — 見出し+内容をそのままヒーローカードの背景に直接置く(入れ子のカード感を出さない)。
- ヒーローカード自体はKPI等の詳細データを内包しない。詳細はヒーローカードの外側・下に別セクションとして続ける(§2.2)。
- 例: `NEO TODAY`(拠点ダッシュボード)、将来的に `SALES TODAY`(営業管理)・`FRONT TODAY`(フロント)等、モジュールごとに同じ構造で用意してよい。ただし未実装のモジュールに空の「◯◯ TODAY」だけを先行して出さない(ERP開発憲章第6条: 実装が伴わないうちは機能があるように見せない)。

### 2.2 KPIグリッド

「○○ TODAY」ヒーローカードの外側・直下に置く、独立した個別カードの並び。

- 各セルは個別に `background:#0F2A4D; border:1px solid #1A2A4A` を持つ(ヒーローカードに内包しない、§2.1参照)。
- 構成: アイコン → ラベル(2行分の `min-height` を固定し、文字量に関わらず数値の開始位置を揃える) → 数値(+単位) → 補足値(稼働率等、無い場合も同じ高さの空要素を描画して段差を作らない)。
- レスポンシブ: PC(≥1180px)4列 / タブレット(760〜1180px)2列 / スマホ(<760px)1列。

### 2.3 二分割パネル(優先度リスト / インサイト)

KPIグリッドの下に置く、左右2カラムの情報パネル。

- 左: 「今日やるべきこと」相当。見出しに💡等の絵文字、右上に「もっと見る ›」。番号は塗りつぶしゴールドの丸+ネイビーの太字数字(Apple Reminders風)。優先度バッジは 緊急/高/中/低 の4段階、それぞれ専用色。
- 右: 「NEOからのお知らせ・提案」相当。見出しに🤖等の絵文字。内部をお知らせ📢／AI分析・提案📈／AI予測📊の3セクションに分け、各セクション見出しに絵文字+色分け。

### 2.4 クイックメニュー(ランチャー)

ページ最下部、そのモジュールでよく使う機能への近道。`ModuleGrid`(会社ホーム等で使う説明文付きの大きいタイル)とは別の、アイコン+ラベルのみの密なグリッド(`src/ui/ModuleLauncher.jsx`)。個別カード背景・ホバー時のみゴールドの縁取り。全項目を並べる必要はなく、モジュールごとによく使う項目を厳選してよい(全項目は左サイドバーから引き続き到達できる)。

### 2.5 サイドバー

- 最上部にそのブランドの正式ロゴ(現状の約1.5倍サイズ、約114px)。クリックでそのブランドのホームへ。
- ロゴの下に、常時表示の「ダッシュボード」等のホーム項目を1つ固定。
- その下にカテゴリーごとに開閉可能なグループ(見出しはゴールド+下線、クリックで開閉)。
- 個々の項目はアイコン+ラベル、選択中は左端ゴールドのインジケーター。

### 2.6 ヘッダー

- 左: パンくず。現在地のみゴールド背景+ネイビー文字の pill チップ、それ以外はグレーの通常文字。
- 右: 通知ベル・ユーザー情報(アバター+名前+ロール+シェブロン)。**常時の背景・枠線は持たせない** — ネイビーの地に直接アイコンを置き、ホバー/タップ時のみ薄い背景が付く。

### 2.7 NEOチャット(全画面共通)

右下固定。閉時はNEO+短いヒントテキストのカード、開くとチャットパネル。ヒントテキストは状況に応じて複数パターンからランダム表示してよい。既存の `src/ai/DaiChat.jsx` をそのまま全画面で共有する(画面ごとに作り直さない)。

## 3. ブランド切替のルール

ロゴと名称だけが `useBrand()` / `BrandProvider` 経由で画面ごとに切り替わる。それ以外(配色・カード・余白・アイコン・フォント・NEO・アニメーション)はブランドに関わらず完全に同一とする。

- 大栄商事株式会社トップ(会社全体ポータル): 大栄商事の正式ロゴ。
- リコホテル三国(および将来追加されるホテル): 各ホテル専用の正式ロゴ。
- NEOの胸バッジ: 本来はブランドごとの単体アイコン(現在は大栄商事の「A」マークで固定、リコホテル用「R」単体アイコン素材がまだ無いため — 既知の未対応事項)。

## 4. レスポンシブ基準

| ブレークポイント | 目安 | KPIグリッド |
|---|---|---|
| PC(Mac/Windows) | ≥1180px | 4列 |
| タブレット横 / iPad横 | 760〜1180px程度 | 2列(現状は4列のまま。タブレット横は追加検証が必要) |
| タブレット縦 / iPad縦 | 760〜1180px | 2列 |
| スマホ | <760px | 1列 |

サイドバーは `C.breakpoint.md`(768px)未満でオフキャンバスドロワーに切り替わる(`src/layout/NavSidebar.jsx`の既存挙動、本デザインシステムでも維持)。

## 5. 今後の適用方針

本デザインシステムは今後追加されるすべての画面(未実装のフロント・清掃・朝食・夕食・駐車場・設備管理・シフト管理・入金管理・キャッシャーレポート・購入申請・経費精算・書類管理・NEO設定を含む)の標準とする。既存の稼働中画面(会社ホーム・営業管理・案件管理・契約管理・社員情報・拠点設定等)への展開は、画面ごとに個別の移行提案(ERP開発憲章第28条の手続き)を経て順次行う — 一度に全画面を書き換えることはしない。
